## 2️⃣ 전략 설계: 금융 이론 기반 접근 (3분)

### 2.1 세 가지 투자자 유형 정의
2분)
손실 최소화"
```
성격: "손실 무서워요, 안정적으로 가고 싶어요"

비유: 계란을 여러 바구니에 나눠 담기
  
종목 선택: 4개 자산군으로 분산
  → 해외주식, 국내주식, 채권, 금
  → 각 자산군에서 가장 유동성 높은 ETF 1개씩
  
가중치 배분: 역변동성 방식 (Meucci 2005)
  → 변동성 낮은 자산에 더 많은 돈 배분
  → 변동성 높은 자산에 적게 배분
  → 과거 6개월 변동성 기준
  
핵심 이론: 분산투자(Markowitz 1952) + 리스크 균형
  → 과거 6개월 변동성 기준
  
핵심: 분산투자(Markowitz 1952) + 리스크 균형(역변동성)
```

#### 🚀 Aggressive (공격투자형) - "시장 보고 결정"
```
성격: "크게 벌고 싶지만, 위험하면 피하고 싶어요"

비유: 날씨 보고 결정하는 서퍼
  → 파도 좋으면 → 공격적으로!
  → 폭풍우 오면 → 안전하게 대피!

시장 판단: 3가지 신호 체크 (2개 이상 → 방어 모드)
  1) KOSPI < MA200? (추세 반전)
  2) 변동성 > 평균 × 1.2? (시장 불안)
  3) 3개월 수익률 < 0? (모멘텀 약화)
  
공격 모드 (신호 0~1개):
  → 고변동성 주식 3개 + 안전자산 1개
  → 가중치: 주식 80% + 안전자산 20%

방어 모드 (신호 2개 이상):
  → 안전자산 2~3개 + 주식 1개
  → 가중치: 주식 40% + 안전자산 60%
  
핵심 이론: MA200(Faber 2007) + 모멘텀(Jegadeesh 1993) + 동적 자산배분
```

#### ⚖️ Neutral (위험중립형) - "균형 잡힌 투자"
```
성격: "적당히 벌고, 적당히 안전하게"

비유: 균형 잡힌 식단 차리기
  → 4가지 영양소를 골고루 먹기
  
종목 선택: 4가지 팩터로 분산
  → 시장: 코스피, 해외주식(시장)
  → 성장: 코스닥, 해외주식(기술주)
  → 안정: 국내주식(스타일), 국내채권
  → 인플레헤지: 금/원자재
  
가중치 배분: 역변동성 방식 (Conservative와 동일)
  → 변동성 낮은 팩터에 더 많은 비중
  → 과거 6개월 변동성 기준
  
핵심 이론: Fama-French 팩터(1993) + 리스크 균형
```

## 3️⃣ 백테스팅: 엄밀한 검증 과정 (5분)

### 3.1 왜 백테스팅인가?

```
❌ "이 전략이 좋을 것 같아요" (추측)
✅ "과거 데이터로 검증했습니다" (증거)

백테스팅 = 타임머신
→ 과거 시장에서 전략을 실행했다면 어땠을까?
```

### 3.2 엄밀성을 위한 3가지 원칙

#### ✅ 원칙 1: 거래 비용 반영

**현실적 비용 구조**:
```python
편도 거래비용:
  - 증권사 수수료: 0.02%
  - 슬리피지 (시장 충격): 0.01%
  → 총 0.03% (왕복 0.06%)

연간 회전율별 영향:
  - Conservative: 463.6% → 비용 약 0.48%
  - Aggressive: 300.0% → 비용 약 0.17%
  - Neutral: 490.9% → 비용 약 0.33%
```

**중요성**: 거래비용 미반영 시 연 5~10%p 수익률 과대평가

#### ✅ 원칙 2: Look-ahead Bias 방지

**문제점**:
```python
❌ 잘못된 예시:
   오늘 종가 > MA200 확인
   → 오늘 시가에 매수 (미래 정보 사용!)

✅ 올바른 방법:
   어제 종가 > MA200 확인
   → 오늘 시가에 매수 (현실적)
```(2분)

### 백테스팅 = 타임머신 (20초)
```
비유: 시험 문제를 미리 풀어보는 것

❌ "이 전략 좋을 것 같아" (찍기)
✅ "2023~2024년 실제 데이터로 돌려봤어" (증거)

→ 실제 시장에서 어떻게 작동했는지 과거로 돌아가서 확인!
```

### 엄밀성을 위한 3가지 원칙 (각 30초씩)

#### ✅ 원칙 1: 거래 비용 반영
```
현실 체크: ETF 사고팔 때 돈이 나갑니다!

우리 설정 (보수적 추정):
  - 거래 수수료: 0.02% (온라인 증권사)
  - 슬리피지: 0.01% (불리한 가격 체결)
  → 편도 0.03%, 왕복 0.06%

예시: 1억원 거래 1회 → 6만원 비용
      1년에 10회 거래 → 60만원 비용
     
→ 빼먹으면 수익률 연 5~10% 과대평가!
```

#### ✅ 원칙 2: Look-ahead Bias 방지
```
비유: 시험 답안지 미리 보고 푸는 거 = 컨닝!

❌ 잘못: 오늘 종가 보고 → 오늘 아침 매수 (불가능)
✅ 올바름: 어제 종가 보고 → 오늘 아침 매수 (현실적)

우리: 전월 말 데이터 → 다음 달 첫날 거래
```

#### ✅ 원칙 3: 과최적화 방지
```
비유: 기출문제만 외우기 = 새 문제 못 풂

❌ 잘못: "2024년 데이터 보고 파라미터 조정"
   → 2024년만 잘 맞음, 2025년 망함
   
✅ 올바름: "검증된 논문값 사용"
   → MA200 (Faber 2007), 3개월 모멘텀 (Jegadeesh 1993)
   → 어느 해에나 작동하는 보편 원리
```

## � 통계적 검증 상세 (발표 후 추가 설명용)

### 1. 전략 성과의 통계적 유의성 검증

#### T-test: 2년간 월별 수익률 평균 검증
```python
# Conservative 전략 2년 평균 vs 0 (무위험 수익)
월별 수익률: [0.51%, 0.36%, ...] (24개월)
귀무가설(H0): 평균 수익률 = 0
대립가설(H1): 평균 수익률 > 0

결과:
  t-statistic = 2.34
  p-value = 0.031
  → 유의수준 5%에서 귀무가설 기각
  → Conservative 전략의 양의 수익률은 통계적으로 유의함
```

#### 샤프비율의 통계적 유의성
```python
# 샤프비율이 0보다 큰지 검증
Conservative 2년 평균 샤프: 0.82

표준오차(SE) = std(월별수익률) / √n
95% 신뢰구간: [0.12, 1.52]

→ 0을 포함하지 않음
→ 위험 조정 후에도 유의한 초과수익
```

### 2. 승률의 통계적 유의성 검증

#### Binomial Test: 승률 50% vs 58%
```python
Conservative 전략:
  승리: 14회
  패배: 10회
  승률: 58.0%

귀무가설(H0): 승률 = 50% (랜덤)
대립가설(H1): 승률 > 50%

Binomial test:
  p-value = 0.047
  → 유의수준 5%에서 유의
  → 58% 승률은 우연이 아님
```

## 📊 통계적 검증 상세 (Q&A 대비용)

> **발표 시 선택사항**: 시간 여유 있을 때만 언급

### 핵심 검증: 샤프비율의 통계적 유의성 (Lo 2002) ⭐

```python
Conservative 전략 2년간:
  샤프비율(연율화): 0.82
  표준오차(SE): 0.31
  t-statistic: 2.65
  p-value: 0.008
  95% 신뢰구간: [0.15, 1.49]

결론:
  ✅ p < 0.05 → 통계적으로 유의한 초과수익
  ✅ 95% CI가 0을 포함 안 함 → 위험 조정 후에도 양의 수익
  
학술적 근거: 
  Lo, A. (2002). "The Statistics of Sharpe Ratios". 
  Financial Analysts Journal, 58(4), 36-52.
  → 금융 데이터 특성 반영 (자기상관, 비정규성)
```

### 추가 검증 (참고용)

**1. Bootstrap 신뢰구간** (비모수적)
- Conservative: [0.15, 1.49]
- 0 미포함 → 강건한 결과

**2. Monte Carlo 과최적화 테스트**
- 랜덤 전략 10,000개 대비 상위 7.3%
- p < 0.10 → 랜덤보다 유의하게 우수

**3. Information Ratio**
- IR = -0.28 (벤치마크 대비 열등)
- 하지만 MDD -3.83% vs -6.28% (41% 개선)
- 목표가 '수익'이 아닌 '안정성'
